# Typecheck Agent Template

Use this template when building a TypeScript type-checking agent for a specific project. The workflow is standard — only project context and canonical type sources differ.

Agents built from this template invoke `/typecheck-workflow` skill for the standard process. They add only what is project-specific: commands, generated file locations, and common error patterns.

---

## Frontmatter

```yaml
---
name: <project>-typecheck
description: <Project> TypeScript type-checking agent. Runs tsc --noEmit, interprets errors in the context of generated types and Zod schemas, and proposes typed fixes. Use when type errors block a build or when hardening types.
tools: Bash, Read, Glob, Grep
model: inherit
---
```

---

## Body structure

### 1. Identity (1–2 lines)
What project this checks. What TypeScript configuration is in play (strict, noUncheckedIndexedAccess, etc.).

### 2. Project context

```markdown
## Project context

- Working directory: `<path>`
- Type-check command: `<pnpm typecheck | npx tsc --noEmit | ...>`
- tsconfig: `<path/to/tsconfig.json>`
- Strict mode: yes/no
- Generated files (never hand-edit): `<list>`
```

### 3. Workflow
Reference `/typecheck-workflow` skill. Add project-specific overrides:

```markdown
## Workflow

Follow the standard `/typecheck-workflow`. Project-specific:

- Pipe output: `<command> 2>&1 | head -200`
- Canonical type sources for this project: see below
```

### 4. Canonical type sources
Where to find types instead of guessing or casting. Reference `CANONICAL_SOURCES.md` for the full table:

```markdown
## Canonical type sources

See `CANONICAL_SOURCES.md` for the full lookup table.

Quick reference for this project:
- Generated DB types: `<path>`
- Zod schemas: `<path pattern>`
- External API types: `<path>`
```

### 5. Generated files — never hand-edit
List the files that are auto-generated. If tsc complains about them, fix the generator or migration, not the file:

```markdown
## Generated files — read-only

- `<path>` — generated by `<tool>`, regenerated by `<command>`
```

### 6. Common error patterns

```markdown
## Common patterns and fixes

- **`Property X does not exist on type Y`** → look up Y in canonical sources; the correct type is likely already defined
- **Non-null assertion `!`** → guard with `if (value)` instead; treat the null case
- **`as any`** → never use unless truly unavoidable; find the real type instead
- **`Type 'X' is not assignable to type 'Y'`** → check if Y has a union; add the missing case
```

---

## What NOT to include

- ❌ The full typecheck workflow — it lives in `/typecheck-workflow` skill
- ❌ Tool guardrails — see `TOOL_RULES.md`
- ❌ Canonical source tables — see `CANONICAL_SOURCES.md` (add a project-specific quick ref, not the full table)
- ❌ Suggestions to use `as any` or `as unknown` as a solution
